# 作図仕様（シーケンス図スタイル）

本アプリは、参照サンプル（実在の電気図面「コンプレッサー遠方運転盤」の制御回路図／`sample/ex1.png`）の流儀に準拠する。
作図の**記法**はラダー図的（グリッド上の記号配置）だが、記入・運用の**流儀は「シーケンス図（展開接続図）」スタイル**を採る。

## 構成要素
- **縦型2母線 + 横回路**: 左右の縦母線の間に各回路を横に渡す。母線名は**固定でなくユーザーが選択・設定できる**（例 `N24`/`P24`、`N`/`P`）。
- **実機器名での識別（アドレスではない）**: `CR11`（リレー）、`PB`（押ボタン）、`PL(G)/PL(R)`（表示灯）、`SS`（セレクトスイッチ）等、部品リストの型式に対応する実名で管理。
- **配置慣習**: コイル／負荷は右母線側、接点群は左側。
- **クロスリファレンス**: コイルと接点の所在を相互参照。**参照は機器名＋回路番号**で行う（サンプルの「ページ+ステップ」方式は不採用）。図面下部にコイル別の使用箇所一覧表を自動生成。
- **設置場所のグルーピング枠（点線）**: `中継ボックス` `MR盤` `ポンプ盤` 等、物理的にどの盤・箱に収まるかを枠で表現。
- **自己保持**: SET/RST 命令ではなく、補助リレー接点による実リレー回路で構成。

## 番号体系（いずれもアプリが自動採番）
- **回路番号**: 横の回路線ごとに付与する**図面全体を通した連番**。1出力が必ず1本とは限らず**中間に2〜3本の並列線**を持つことがあるため、出力単位ではなく「横の回路線」単位で振る。
- **線番号（ワイヤ番号）**: 電気的に連続した配線（ネット）ごとの番号（[data-model.md](data-model.md) 参照）。採番ルーチンは下記「線番採番ルーチン」で確定。
- **行番号（ステップ番号）は不採用**: サンプル左端番号は「ページ+ステップ」のクロスリファレンス用（例 `712`＝7ページ12ステップ）だが本アプリは使わない。グリッドの行は内部座標としてのみ保持。
- **採番方式（確定）**: 接頭辞なしの単純連番。番号追加時は**自動で順送り再採番**し、連番・重複なしを保つ。

## 線番採番ルーチン（確定・サンプル `sample/sample-CLD.pdf` 準拠）
参照サンプル（`sample/sample-CLD.pdf`、線番 `1`〜`19`／母線 `R200`・`S200`）の流儀を正典とする。

1. **母線ネットは番号でなく名前**: CP二次側など各母線ネットは母線名（既定 `BusConfig` の `LeftName`/`RightName`、例 `R200`/`S200`）で識別し、連番 1,2,3… の対象外とする。
2. **連番は「内部ネット」だけに振る**: 母線に直結しないネット（機器と機器の間のネット）にのみ `1` から連番を付与する。母線直結セグメント（左母線→最初の接点、コイル／負荷→右母線）は番号を振らず、母線ネット名に属する扱い。
3. **採番順＝読み順**:
   - 回路（横の回路線 `CircuitLine`）を**上→下**。
   - 各回路内では、まず**主線（コイル／負荷へ至る主経路）を左→右**に辿って番号を振る。
   - 続いて**並列・分岐枝**を上→下／左→右の順で辿る。
   - 例（CR5回路）: 主線 `7`(CR1–CR3) `8`(CR3–コイル) → 並列枝 `9`(CR2–SOV1-ON) `10`(SOV1-ON–SOV1-OFF)。
4. **1ネット1番号**: 電気的に同一のネットには番号は1つだけ。自己保持枝など既存ネットへ**合流する枝は新番号を振らず**、合流先ネットの番号を共有する（サンプルの CR4/CR5/CR6 自己保持枝は番号を持たない）。
5. **自動再採番**: 要素・配線の追加／削除時は上記の読み順で**自動で再採番**し、連番・重複なしを保つ（手動固定の要否は [todo.md](todo.md)）。

## 作画モードの入力方式
- 図面は**均一グリッド**で構成し、各セルに要素を配置。**セルサイズは全要素で同一**。
- ただし**1ユニットで複数セルを占有する要素**がある（例: セレクトスイッチ=3セル、a/b接点・コイル=1セル）。種別ごとに占有セル数を定義し、配置時に確保。
- ツールバーの**ボタンで要素種別を選択**し、グリッドのセルへ**挿入**する。
- **挿入時**: 既設要素は**右へシフト**してスペースを確保。右シフトで**行末（母線）からはみ出す場合は警告を表示して挿入を拒否**（自動の列拡張・折り返しはしない）。
- **行をまたぐ配置は基本 NG**（1ユニットは同一行内）。並列回路は要素の行またぎではなく**縦コネクタ**で表現。
- **分岐**: 配線が分岐・接合する箇所に**黒いドット（●）= 接合点**を記入して接続を明示。ドットの無い交差は非接続。
- **端子台**: 配線が外部装置と接続する箇所に**端子台記号**を記入（黒ドット＝分岐 とは別の意味）。サンプルでは二重丸（◎）で描かれているが**暫定表記**で実運用時に別図形へ変更予定。データモデルは見た目に依存しない「端子台」種別として持つ。

## PLCラダーとの対比
| 観点 | PLCラダー | 本アプリ（シーケンス図／展開接続図） |
|---|---|---|
| 母線 | 左右の論理母線 | 実電源（縦2母線・名称設定可） |
| 要素の識別 | アドレス（X/Y/M） | 実機器名（`CR11`, `PB`, `PL`…） |
| 接点記号 | 抽象シンボル `—[ ]—` | 実デバイス記号（押ボタン・リレー接点） |
| 参照 | アドレスで暗黙リンク | クロスリファレンス（機器名＋回路番号）＋一覧表 |
| 物理情報 | なし（論理のみ） | 設置場所枠・実配線に対応 |
| 動作モデル | スキャン（上→下の逐次） | 実リレーの**電気的同時動作** |
