# UI 刷新プラン — GX Works3 風レイアウト

> **目的**: 三菱 GX Works3 の操作感に近い UI にリファクタリングする。
> **方針**: GX Works3 の外観・操作体系を参考にしつつ、本アプリ固有の機能（有接点リレー・シーケンス図スタイル）に合わせた形で採用する。
> **実装ステータス**: P1〜P6 すべて完了（P3: 下部出力パネル 2026-06-20 / P4 プロパティタブ完全実装 2026-06-20）。
> 関連: [operation-features-plan.md](operation-features-plan.md) / [next-plan.md](next-plan.md)

---

## GX Works3 主要 UI 要素と本アプリの対応

| GX Works3 要素 | 現在の状態 | 本プランでの扱い |
|---|---|---|
| メニューバー | あり（ファイル/編集/図面/表示/ヘルプ） | 強化・項目整理 |
| 標準ツールバー（アイコン1行・薄型） | なし | 新規追加 |
| ラダー図ツールバー（縦サイドパレット） | 横水平バー（要素多）→ 縦整理 | **縦ツールパレット（左端）に移植** |
| ナビゲーションウィンドウ（左ドック） | なし（TabView のみ） | **追加: プロジェクトツリー** |
| 作図エリア（中央） | Win2D Canvas | 現状維持 |
| クロスリファレンス/プロパティ（右ドック） | 機器表パネル（単一） | タブ切り替え（機器表/Xref/プロパティ） |
| 出力ウィンドウ（下ドック） | なし | **追加: DRC/検索結果タブパネル** |
| ステータスバー | あり（簡易） | 強化（モード/行列/回路番号） |

## 変更項目（優先度順）

### P1: 縦ツールパレット（左サイドバー）[優先度: 高]

**概要**: 現在の横 ToolStackPanel を廃止し、左端に幅 60px 程度の縦パレットを配置する。

**GX Works3 との対応**:
- GX Works3 は上部水平ツールバーと左縦ツールバーの2段構成が多い
- 本アプリでも「接点系 / 出力系 / その他 / 編集」を縦に並べる

**レイアウト（縦パレット）**:
```
┌────────┐
│ ↖ 選択 │  ← 常に一番上
├────────┤
│ ─a─ NO │
│ ─b─ NC │
│ ─pb─NO │
│ ─pb─NC │
│ TaNO  │
│ TaNC  │
├────────┤
│ (O) コイル│
│ ◎ 表示灯 │
│ ┤T├ タイマ│
│ □ 端子 │
├────────┤
│ ║ 分岐 │
│ □ 枠   │
├────────┤
│ その他▼│
└────────┘
```

**実装要点**:
- `Page.Resources` の `ToolBtnStyle` (RadioButton) はそのまま流用
- `ToolStackPanel` の `Orientation="Horizontal"` → `Orientation="Vertical"` に変更
- 各ボタン幅を `Width="60"` 固定、アイコンと文字を縦並び
- メイングリッドの左列として配置（`Grid.Column="0"`、幅 `Auto`）
- テストモード時は非表示またはグレーアウト

**影響ファイル**:
- `MainPage.xaml`: ToolStackPanel を左列の `Border` 内の縦 `StackPanel` に移動
- `MainPage.xaml.cs`: 変更なし（イベントハンドラは流用）

---

### P2: ナビゲーションウィンドウ（プロジェクトツリー）[優先度: 高]

**概要**: 縦パレットの右隣、または上部に折りたたみ可能な左ドックパネルを追加する。

**GX Works3 との対応**:
- GX Works3 の「ナビゲーションウィンドウ」= 左ドックの TreeView
- 「プロジェクト」「デバイス」「接続先」のルートノードがある

**本アプリでの TreeView 構成**:
```
▼ プロジェクト
   ▼ シート
      ├ 1. Sheet1
      ├ 2. Sheet2
      └ [+ 新規追加]
   ▼ デバイス一覧（機器名アルファベット順）
      ├ CR11
      ├ CR12
      └ ...
   ▼ 図面情報
      ├ 図番: 001
      └ 作成日: ...
```

- シートノードをクリック → 対応シートに切り替え（現在の TabView と連動）
- デバイスノードをクリック → 機器表パネルをフォーカス・対象行をハイライト
- 折りたたみトグルボタンで表示/非表示（幅 200px / 0px）

**実装要点**:
- WinUI 3 の `TreeView` コントロール
- `TreeViewNode` のデータは `MainPage.xaml.cs` 側でシートリスト変更時に更新
- 折りたたみ: `Width="200"` と `Width="0"` の間で `Visibility` 切り替え、またはスプリッター

**影響ファイル**:
- `MainPage.xaml`: 左ドック列 (`TreeView` + 折りたたみボタン) を追加
- `MainPage.xaml.cs`: `BuildNavigationTree()` メソッド + シート選択イベント連動

---

### P3: 下部出力ウィンドウ[優先度: 中]

**概要**: キャンバスの下にドッキングパネルを追加し、DRC・検索結果・接続検査結果を一覧表示する。

**GX Works3 との対応**:
- GX Works3 の「出力ウィンドウ」= 下部ドック、コンパイル/検索/クロスリファレンスのタブ

**本アプリでの構成**:
```
┌ DRCエラー ┬ 検索結果 ┬ 接続検査 ┐
│ P5: 行3 コイルが2重定義 (CR11)    │
│ P3: 行5 横配線に要素なし          │
└──────────────────────────────────┘
```

- 「DRCエラー」タブ: DesignRuleCheck の結果をリスト表示、行クリックで問題箇所にジャンプ
- 「検索結果」タブ: 現在の FindBar の結果を下パネルへ移動
- 「接続検査」タブ: 接続検査 ON 時のネット警告一覧

**実装要点**:
- `GridSplitter` でキャンバスと下パネルの高さ調整可能にする
- 高さ `100〜150px` デフォルト（折りたたみボタンで高さ 0 へ）
- `TabView` または `TabControl` でタブ切り替え
- DRC 実行は手動ボタン（「DRC実行」）または保存時に自動実行

**影響ファイル**:
- `MainPage.xaml`: 下部パネル列と `GridSplitter` を追加
- `MainPage.xaml.cs`: DRC実行 → `ListView` に結果追加、行クリック → シート切替 + スクロール

---

### P4: 右ドックパネルのタブ化[優先度: 中]

**概要**: 現在の「機器表パネル」を拡張し、タブで切り替え可能な右ドックパネルにする。

**タブ構成**:
| タブ名 | 内容 |
|---|---|
| 機器表 | 現在の `DevicePanel`（DeviceTable 一覧） |
| クロスリファレンス | デバイスごとのコイル/接点参照一覧 |
| プロパティ | 選択中の要素のプロパティ（機器名・コメント・Params） |

**GX Works3 との対応**:
- GX Works3 右側の「インテリジェント機能ユニット設定」「デバイスコメント」等のパネルに相当

**実装要点**:
- 既存の `DevicePanel Border` を `TabView` 内に移動
- 「プロパティ」タブは選択中要素が変わる度に自動更新
- 幅は現在の 210px を維持（ユーザーがリサイズ可能にするなら `GridSplitter`）

---

### P5: 標準ツールバー（薄型1行）[優先度: 低]

**概要**: メニューバーの直下に GX Works3 の「標準ツールバー」相当の薄いアイコン行を追加する。

**内容**（アイコンボタン、16px 程度）:
```
[新規] [開く] [保存]  |  [元に戻す] [やり直し]  |  [PDF出力]  |  [DRC実行]  |  [全体表示]
```

**実装要点**:
- WinUI 3 の `CommandBar`（`AppBarButton` 使用）が最も適している
- 現在のツールバー（Row1）の上に Row0.5 として追加するか、MenuBar と合体させる
- アイコンは `FontIcon`（Segoe Fluent Icons）で統一

---

### P6: ステータスバー強化[優先度: 低]

**GX Works3 との対応**（ステータスバー右側に固定情報）:

現在: `[ツール表示] [StatusPos: 行・列] [StatusTitle: ファイル名] [接続検査ON/OFF]`

追加項目:
| 項目 | 内容 |
|---|---|
| 編集モード | 「作画」 / 「テスト」 |
| 回路番号 | 選択行の回路番号（現在未表示） |
| シート枚数 | 「3シート / 5シート」 |
| 変更有無 | 「変更済み●」 / 「保存済み」（Dirty フラグ） |

## 実装難易度

| 項目 | 難易度 | 実装量 | 備考 |
|---|---|---|---|
| P1: 縦ツールパレット | 低 | 小 | XAML 整理のみ |
| P2: ナビゲーションツリー | 中 | 中 | TreeView + シート連動 |
| P3: 下部出力パネル | 中 | 中 | GridSplitter + DRC連携 |
| P4: 右パネルタブ化 | 低 | 小 | 既存 DevicePanel の整理 |
| P5: 標準ツールバー | 低 | 小 | CommandBar 追加 |
| P6: ステータスバー強化 | 低 | 小 | テキスト追加のみ |

**推奨実装順序**: P1 → P4 → P6 → P5 → P2 → P3

## 未確定事項（ユーザー判断が必要）

1. **縦ツールパレットの幅とボタンサイズ**: 60px 幅のアイコン+文字 or 40px アイコンのみ（ホバーで名前表示）
2. **ナビゲーションツリーとシートタブの関係**: 両方残す or ツリーに一本化する
3. **下部パネルのデフォルト表示**: 常時表示 or 初期非表示（DRC実行時に自動展開）
4. **テーマ色**: GX Works3 はライトテーマ固定。本アプリも固定 or Win11 システムテーマ追従
5. **ドッキングウィンドウの独立化**: 右パネル・下パネルを別ウィンドウに切り離せるか（WinUI3 の制約あり）
