# 総合テストプラン（パッケージング前）

> 作成 2026-06-22。配布（MSIX 等）の前に全機能を網羅的に検証するためのプラン。
> 関連: [todo.md](todo.md)（残=配布のみ）、各 `docs/` 正典。
> **方針**: ロジックは自動テスト（xUnit）で担保済み。WinUI3 の UI 操作は自動化困難なため**実機チェックリスト**で網羅する。

## 0. 前提・環境
- ビルド: `dotnet build src/GuiEcad.App/GuiEcad.App.csproj -r win-x64`（0警告0エラーを維持）
- 自動テスト: `dotnet test GuiEcad.sln`（現状 164 件・全合格）
- 実機: `dotnet run --project src/GuiEcad.App`
- 確認用サンプル: 任意の既存 `.gcad`（後方互換確認用）と、その PDF 出力（比較用）を各自用意する
- ライト/ダーク両テーマで主要画面を確認すること。

## 2. 手動・実機テスト（チェックリスト）
> 各項目: 操作 → 期待結果。P1=リリース必須 / P2=できれば。

### 2.1 起動・基本（P1）
- [ ] 起動時に空シートが1枚（サンプル回路を出さない）。
- [ ] 作画/テストのモード切替でステータスバー表示が変わる。ポインタ状態がリセットされる。
- [ ] ウィンドウタイトルが「（無題）- GuiEcad」、編集後に `*` が付く。

### 2.2 作画モード・制御回路（P1）
- [ ] a接点/b接点/押釦/タイマ接点/コイル/表示灯/端子 を配置できる。
- [ ] 縦コネクタ（分岐）で並列接続を作れる。
- [ ] 配置で線番号・回路番号が自動採番され、左に行番号が出る。
- [ ] 左母線左の行番号（`ShowRowNumbers`）が 1 始まりで表示。

### 2.3 主回路モード・三相動力（P1）
- [ ] 主回路モード ON でグリッド表示・自由直線ツールが使える（左右母線/自動横配線は出ない）。
- [ ] 自由直線で母線 R/S/T（縦3本）を引ける（格子点スナップ）。
- [ ] ブレーカ(NFB/MCCB/ELB)・電磁接触器主接点(3P)・サーマル(3P)・モータ を縦/横で配置。
- [ ] ブレーカの型（NFB/MCCB/ELB）をプロパティで切替→ラベルが変わる（Undo 可）。
- [ ] 接続点「点(●)」を交点に配置/選択/削除できる。
- [ ] 配置プレビュー（マウス追従ゴースト）が出る。
- [ ] 主回路記号がネットリスト/DRC/シミュレーションに影響しない。

### 2.4 編集操作（P1）
- [ ] 選択ツールで単一選択・範囲選択（青破線）。
- [ ] コピー/貼り付け（Ctrl+C/V・マウス位置へ貼付）、範囲コピペ。
- [ ] 要素・枠・自由直線の移動。
- [ ] Undo/Redo が全操作で一貫（配置・削除・移動・採番・プロパティ・行列増減）。
- [ ] 行/列の追加・削除（末尾・選択位置）。

### 2.5 機器・機器表・CR（P1）
- [ ] 機器名のインライン入力・プロパティでの変更（同名は連動）。
- [ ] 機器表タブで型式/メーカー/数量を編集（BOM）。
- [ ] クロスリファレンスが正しく出る（コイル↔接点）。CR 専用ページ。

### 2.6 枠（設置場所）（P2）
- [ ] 枠の作成（自由・mm 連続）・ラベル編集・移動・線種変更。ヒットテストが面積最小優先。

### 2.7 プロパティパネル（P1）
- [ ] SelectSwitch ノッチ位置・Timer 設定時間を NumberBox で編集（Undo 可）。
- [ ] 機器名・コメントを全要素共通欄で編集。

### 2.8 ナビツリー（P1）
- [ ] シート追加/削除/リネーム、選択でキャンバス切替、新規シートの母線名デフォルト。

### 2.9 検索・置換（P2）
- [ ] Ctrl+F で検索バー、機器名検索でヒット移動、置換。

### 2.10 DRC・接続検査（P1）
- [ ] 接続検査で未接続・二重コイル・接点数超過等を検出し、下部出力パネルに一覧。

### 2.11 テストモード（P1）
- [ ] 接点 ON/OFF でコイル・表示灯が点灯（connectivity＋不動点）。
- [ ] 自己保持・インターロックが期待通り。発振回路で発振検出。
- [ ] タイマの時限（Tick 進める）が動作。

### 2.12 自作パーツ・ライブラリ（P2）
- [ ] PartEditor で作図・編集（スナップ 1/16・回転・楕円弧）。
- [ ] `.gcadparts`/`.gcadpart` のエクスポート/インポート。

### 2.13 ツールパレット フロート/ドック/吸着（P1）
- [ ] 左ドック⇄フロート切替。タイトルバードラッグで移動。
- [ ] 右パネル手前でクランプ（裏に潜って消えない）。
- [ ] 上端/下端へドラッグで吸着プレビュー表示→横並びツールバー化。
- [ ] 状態・位置が再起動後に復元（`palette-pos.txt`）。リサイズで枠内へ復帰。

### 2.14 永続化（P1）
- [ ] `.gcad` 保存→新規→読込で完全一致（要素・枠・自由直線・点・機器表・設定）。
- [ ] Dirty フラグ（保存/新規/読込でリセット、未保存で終了確認があるなら確認）。
- [ ] 旧バージョンで保存した `.gcad` が開ける（後方互換）。

### 2.15 PDF 出力（P1）
- [ ] 単一ページ/複数ページ分割（28 行）/A4 縦枠/表題欄右下。
- [ ] CR 専用ページ・BOM ページが付く。
- [ ] フォントがゴシック（Yu Gothic）で文字化けなし。主回路の自由直線が余白/表題欄へはみ出さない（縦クリップ）。
- [ ] 旧バージョンの PDF 出力と比較し意図しない差分がない。

### 2.16 テーマ・右クリック（P2）
- [ ] ライト/ダーク切替が全パネル・フロートパレット・吸着帯に反映、再起動で復元。
- [ ] 作画モードの右クリックメニュー（コピー/貼付/行操作/枠線種等）。

## 3. 非機能・回帰（P2）
- [ ] **性能**: 大きめの図面（数百要素・複数シート）で配置/採番/DRC/PDF が実用速度。
- [ ] **エラー処理**: 壊れた/別形式のファイルを開いてもクラッシュせずメッセージ。
- [ ] **長時間操作**: 連続編集・Undo/Redo を多数繰り返してもメモリ/状態が破綻しない。
- [ ] **ビルド出力2系統**: `-r win-x64` ツリーで実機確認（旧バイナリ事故の回避）。

## 4. リリース判定基準（DoD）

- 既知の致命的不具合ゼロ（軽微は既知事項として記録）。
